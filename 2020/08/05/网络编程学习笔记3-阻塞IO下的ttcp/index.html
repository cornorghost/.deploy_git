本节视频里做的阻塞IO下的ttcp实验的代码没有找到在哪，所以实验就暂时忽略

这节讲的是ttcp大文件传输下的阻塞，虽然没有源代码，但是讲的是客户端发送数据大小超过服务端缓冲区大小的情况

服务端伪代码

TcpStreamPtr stream; //tcp流读写器 
while(stream->receiveSome(buf, sizeof(buf))) 接收数据，直到超过buf缓冲大下
{
    stream->sendAll(buf); //缓冲区满后，给客户端发送缓存的数据
}

客户端伪代码

TcpStreamPtr stream; //tcp流读写器 

string message(len, 's');  //待发送的消息，放len个's'
stream->sendAll(message, message.size()); 发送消息，该消息大小应大于服务器缓冲区

vector<char> receive(len); //接受服务器返回的数据
stream->receiveAll(receive.data(), receive.sie()); //发送消息完毕后，开始接收数据

如上图，当客户端发送的数据大于服务器缓冲时，客户端的发送还在进行，而服务端已经开始发送数据给客户端，客户端没进入接收消息的状态，所以服务器等待，停止接收信息，客户端还在一直发送

所以我感觉客户端收发都做线程就可以解决这个问题
